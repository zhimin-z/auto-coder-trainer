## 实验 1：PPO vs GRPO 算法对比

**主题：** 小规模推理任务中的算法效率对比

**核心思想：**
- TinyZero 用 PPO，DeepSeek R1 用 GRPO
- 在相同条件下对比两者效果

**实验设计：**
- 数据集：GSM8K (8K)
- 模型：Qwen2.5-3B
- 对比维度：
  - 训练时间
  - 最终准确率
  - 显存占用
  - 样本效率（达到 60% 准确率需要多少样本）

**预期产出：**
- 对比表格
- 训练曲线图
- 最佳实践建议

**时间成本：** 2-3 天（2 GPU）

**创新点：** 首个小模型（3B）上的 PPO vs GRPO 系统对比

## 实验 2：FP8 量化对 Zero 训练效果的影响

**主题：** FP8 量化对 Zero 训练效果的影响

**核心思想：**
- Zero 训练通常需要大量推理（Rollout）
- 量化可以加速 Rollout，但会影响精度吗？

**实验设计：**
- 数据集：GSM8K
- 模型：Qwen2.5-7B
- 对比组：
  1. BF16 全精度
  2. FP8 Rollout（推理量化）
  3. FP8 端到端（训练+推理量化）
- 测量：
  - 训练速度（samples/sec）
  - 最终准确率
  - 显存占用
  - 推理吞吐量

**预期产出：**
- 量化对 RL 影响的量化分析
- 速度 vs 精度权衡曲线

**时间成本：** 3-4 天

**创新点：** 首个系统研究量化对 RL 推理训练的影响

## 实验 3：数据规模 Scaling Law

**主题：** 推理能力涌现的最小数据量

**核心思想：**
- Zero 训练需要多少数据才能让模型学会推理？
- 找到"推理涌现"的临界点

**实验设计：**
- 数据集：GSM8K 随机采样
- 模型：Qwen2.5-3B
- 算法：GRPO
- 数据量梯度：100, 500, 1K, 2K, 5K, 8K
- 每组训练 15 epochs，测量：
  - 最终准确率
  - 是否出现 CoT（思维链）
  - 收敛速度

**预期产出：**
- 数据量 vs 准确率曲线
- 推理涌现的临界点（例如：>1K 才出现 CoT）

**时间成本：** 4-5 天（6 组实验）

**创新点：** 首次系统研究 Zero 训练的数据效率

## 实验 4：跨任务推理迁移

**主题：** Zero 训练学到的推理能力是否可迁移

**核心思想：**
- 在简单任务（Countdown）学会推理
- 能否迁移到复杂任务（GSM8K）？

**实验设计：**
- 阶段 1：在 Countdown 上 Zero 训练
- 阶段 2：测试在 GSM8K 上的表现
- 对比组：
  1. 直接在 GSM8K 训练（baseline）
  2. Countdown 预训练 → GSM8K 微调
  3. 只用 Countdown 训练，直接测 GSM8K
- 测量：
  - Zero-shot 迁移效果
  - 微调后的提升幅度

**预期产出：**
- 推理能力的通用性分析
- 迁移学习曲线

**时间成本：** 3-4 天

**创新点：** 探索 Zero 训练的推理泛化能力

## 实验 5：奖励函数设计对比

**主题：** 不同奖励信号对推理质量的影响

**核心思想：**
- TinyZero 用 3 档奖励（0, 0.1, 1.0）
- 更细粒度的奖励能否提升效果？

**实验设计：**
- 数据集：GSM8K
- 模型：Qwen2.5-3B
- 奖励函数对比：
  1. Binary（0/1）- 错：0，对：1
  2. Partial（部分奖励）- 格式错：0，格式对但答案错：0.3，答案对：1.0
  3. Process-based（过程奖励）- 每个推理步骤都给奖励，最终答案额外奖励
- 测量：
  - 最终准确率
  - 推理步骤质量
  - 训练稳定性

**预期产出：**
- 奖励函数设计指南
- 最优奖励策略

**时间成本：** 3 天

**创新点：** 系统对比奖励设计对 Zero 训练的影响

## 实验 11：综合 Scaling Law 验证

**主题：** 参数规模 × 数据规模的联合 Scaling 效应

**核心思想：**
- 验证方法是否在不同规模下都有效
- 探索参数和数据的最优配比
- 为大规模训练提供预测

**实验设计：**
- 数据集：GSM8K（分层采样）
- 算法：GRPO（已验证有效的算法）

**实验矩阵：**
```
模型规模 × 数据规模：

           100样本  500样本  1K样本  2K样本  5K样本  8K样本
0.5B        ✓       ✓       ✓       ✓       ✓       ✓
1.5B        ✓       ✓       ✓       ✓       ✓       ✓
3B          ✓       ✓       ✓       ✓       ✓       ✓
7B          ✓       ✓       ✓       ✓       ✓       ✓

总共：4 × 6 = 24 组实验
```

**测量指标：**
- 最终准确率
- 训练收敛速度
- 是否出现 CoT
- 样本效率（每个样本的价值）
- 计算效率（FLOPs vs 性能）

**预期产出：**
- Scaling Law 曲线（参数 vs 数据 vs 性能）
- 最优配比建议（例如：3B 模型需要至少 2K 数据）
- 大规模训练预测（例如：预测 14B 模型需要多少数据）
- 方法有效性验证（是否在所有规模都有效）

**分析维度：**
1. **参数 Scaling：** 固定数据量，看模型规模影响
2. **数据 Scaling：** 固定模型，看数据量影响
3. **联合 Scaling：** 参数和数据同时增长的最优路径
4. **涌现临界点：** 推理能力出现的最小参数+数据组合

**时间成本：** 8-10 天（24 组实验，可并行）

**创新点：**
- 首个 Zero 训练的系统 Scaling Law 研究
- 为工业界提供资源规划依据
- 验证方法的可扩展性

**实用价值：**
- 回答"我有 X 算力，应该用多大模型+多少数据？"
- 预测大规模训练效果
- 判断方法是否值得投入更多资源

## 实验 12：Pass@k 真实能力评估

**主题：** 用 Pass@k 验证 RL 是否真正扩展推理能力

**核心思想：**
- NeurIPS 2025 Best Paper Runner-up 发现：RL 只提高采样效率，不扩展推理能力
- Pass@1 高不代表能力强，Pass@k（大k）才能看出真实边界

**实验设计：**
- 数据集：GSM8K + MATH
- 模型：Qwen2.5-3B
- 对比：Base Model vs GRPO训练后

**评估指标：**
```
Pass@k (k = 1, 5, 10, 20, 50, 100)
- Base Model 的 Pass@100
- GRPO 的 Pass@100
- 关键问题：GRPO 能否超越 Base 的 Pass@100？
```

**预期产出：**
- Pass@k 曲线对比
- 推理能力边界分析
- 验证方法有效性

**时间成本：** 2-3 天

**创新点：** 对标 NeurIPS 2025 顶会方法

## 实验 14：蒸馏 vs RL

**主题：** 蒸馏能否超越 RL？

**核心发现：**
- NeurIPS 论文：蒸馏能引入新推理模式，RL 不能

**实验设计：**
- 方法对比：
  1. GRPO（你的方法）
  2. 从大模型蒸馏（Qwen2.5-7B → 3B）
  3. GRPO + 蒸馏（混合）
- 评估：Pass@k + 推理路径多样性

**预期产出：**
- 蒸馏 vs RL 效果对比

## 实验 21：奖励塑造思考方式

**主题：** 通过奖励函数控制思考的速度和巧妙度

**核心问题：**
- 能否训练"快速思考"？
- 能否训练"巧妙思考"？
- 不同奖励如何影响思考风格？

**实验设计：**

**问题示例：** "125 × 8 = ?"

**奖励方案对比：**

**方案 A：只看结果（baseline）**
```
奖励函数：
- 答案对：+1.0
- 答案错：0

结果：
Model: "<think>1000</think>" (不管过程)
```

**方案 B：奖励步骤数（慢思考）**
```
奖励函数：
- 答案对：+1.0
- 每个推理步骤：+0.1

结果：
Model: "<think>
  125×2=250
  250×2=500
  500×2=1000
</think>" (3步，详细)
```

**方案 C：惩罚冗余（快思考）**
```
奖励函数：
- 答案对：+1.0
- 步骤数惩罚：-0.05 × steps

结果：
Model: "<think>125×8=1000</think>" (1步，简洁)
```

**方案 D：奖励巧妙度**
```
奖励函数：
- 答案对：+1.0
- 使用技巧：+0.3
  (如：分配律、结合律、凑整)

结果：
Model: "<think>
  125×8 = 125×(10-2)
       = 1250-250
       = 1000
</think>" (巧妙方法)
```


**测量指标：**
1. 平均步骤数
2. 思考时间（token 数）
3. 方法多样性
4. 准确率

**实验矩阵：**
```
数据集：GSM8K
模型：Qwen2.5-3B
算法：GRPO

4 种奖励方案 × 测试集
→ 对比思考风格
```

**预期结果：**
- 方案 B：步骤多，详细但慢
- 方案 C：步骤少，快但可能出错
- 方案 D：方法巧妙，准确率高

**巧妙度判断方法：**
```
规则 1：使用数学技巧
- 凑整：125×8 = 1000/8×8
- 分配律：a×(b+c) = a×b + a×c
- 结合律：重新组合

规则 2：步骤效率
- 相同准确率下，步骤越少越巧妙

规则 3：可解释性
- 有清晰的推理逻辑
```

**时间成本：** 4 天

**创新点：** 首次系统研究奖励对思考风格的影响

**实用价值：**
- 快速思考：低延迟应用
- 详细思考：教育场景
- 巧妙思考：展示推理能力
